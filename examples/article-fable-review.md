# 50 美元，买到了什么

凡事要稳，这是我一直信奉的原则。但 6 月 10 日凌晨三点多，我把这句话踩了第一脚。

---

## 凌晨三点：第一次触限

Fable 5 发布了。发布的半小时之内，我已经在终端里输入了 `ultracode 对最近一周的提交进行 code review`，切换到了新模型，准备来一次「浅浅的周度代码审查」。

我不是没想到会贵。Fable 5 的定价是 $10/$50 per million tokens，是 Opus 4.x 的两倍，是 Sonnet 的三倍多。Ultracode 模式会自动写一套调度脚本，拉起几十个并行的 subagents——一次审查扑上来 82 个 agent，9 个维度同时跑，每个都在以 xhigh 推理等级消耗算力。

但我告诉自己：就浅浅看一下，快的。

13 分钟。

系统提示：`You've hit your monthly spend limit`。账户余额：$0.00。

好家伙。13 分钟，把月度用量限额打穿，顺手把账号里剩下的 50 美元全部清零。还好开了月消费上限，还好没有开自动充值。要不然这个故事就更精彩了。

---

## 早上七点：翻「未验证」条目

早上六点多，我打开终端看结果。

不出意外——跑完了一半。审查阶段（9 个维度）全部完成，但验证阶段的 47 个 agent 因为额度耗尽中途死掉了。报告分两块：「已经独立验证确认的」和「审查员原始发现、建议人工确认的」。

我开始翻「未验证但需要重点关注」里的条目。

第三条：reconciler——任务流的后台对账循环——存在一个崩溃链。每轮扫描一旦碰到 `IntegrityError`，session 回滚，所有 ORM 对象进入 expired 状态；然后代码在 except 块里访问 `template.code` 做日志记录，触发懒加载，异步 session 抛出 `MissingGreenlet`；连异常处理本身也崩了，整轮扫描死掉。模板按 id 顺序处理，第一个崩了，后面全部轮不到。

没有独立 agent 去实读代码反驳这条——因为验证 agent 已经死光了。但描述够详细，看完我知道这大概率是真的。

实读确认。代码确实是这样写的。

当天上午修完，6 个高危条目全部走 TDD，464 个测试全过，提交 `e2c2f64`。

---

## 凌晨零点：生产上的实例怎么还没出来

6 月 11 日 00:25，我在后台配了三个任务流模板，发布了，等了老半天，一个实例都没有出来。

SSH 进服务器，`grep '[Reconciler]' server.log`：

```
MissingGreenlet: greenlet.switch to a different greenlet ...
```

隔 5 分钟一条，从 6 月 11 日 00:01 开始，已经循环崩了 8 轮。

和审查报告里的描述一模一样。

对了时间线：

| 时间 | 事件 |
|---|---|
| 6/10 03:13 | 生产部署了包含任务流升级的版本，reconciler 是旧版 |
| 6/10 上午 | 本地修复了 reconciler，提交 `e2c2f64` |
| 6/11 00:10 | admin 发布了 3 个任务流模板 |
| 6/11 00:01 起 | 生产日志每 5 分钟一条崩溃 |

修好的代码在本地，没有部署到生产。

Claude 的语气透着一种难以描述的淡然：「这个 bug 早上已经修复了，只需要部署 hotfix 就行。」

部署，等五分钟。新的报错：

```
[Reconciler] 模板的入口源未注册: person_candidates
```

又一层。入口枚举源靠 import 时的装饰器注册，但这个 import 只在 admin 模板页面的懒加载路径里——reconciler 启动时不 import，注册表是空的。之前每次都会顺手打开 admin，巧合地触发了注册，所以一直没暴露。这次直接在生产配，没有这个「巧合」。

修复一行，再次部署。凌晨一点多，实例出来了。

---

## 11 日上午：第二次触限

凌晨修完生产，睡了几小时，早上继续做昨晚立项的设计系统迁移。

86 个页面全量切换到「经营诊断刊物」视觉体系，这是一次大工程。Phase 1 地基做完，Phase 2 开始精做 9 个锚点屏，Phase 3 派了 4 个并行 agent 同时扫荡长尾页面族群。

启动了，出了门，心想差不多跑完了，下午看结果就行。

回来一看，会话重启了，4 个后台 agent 全部中断。5 小时滚动窗口再次打满——12 分钟。

token 干完了，工作全是半成品。

---

## 那 50 美元值吗

两次触限，一次把月度账单烧光，一次把工作打断。

但那次烧掉 50 美元的 code review，找到了一个真实的 0 级 bug。

那个 bug 当晚在生产上出现了。修复代码早就在本地等着，只差一次部署。

能说什么。值了。

以后切 Fable 加 ultracode 之前，先看一眼账户余额。

凡事要稳一手，我说的。
